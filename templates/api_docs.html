API Endpoints

Session-authenticated REST endpoints for employee CRUD.

Authentication
  • POST /api/login - login with JSON body: username, password
  • POST /api/logout - clear current session
Employees
  • GET /api/employees - list employees (user/admin)
  • POST /api/employees - create employee (admin only)
  • GET /api/employees/<id> - get one employee (user/admin)
  • PUT /api/employees/<id> - update employee (admin only)
  • DELETE /api/employees/<id> - delete employee (admin only)
Sample JSON for create/update
{
  "name": "Alex Doe",
  "email": "alex@example.com",
  "dept": "Engineering"
}
